# Test Design-0001: 적응형 파이프라인 프롬프트 검증

> Source: SDD-0001
> Date: 2026-03-10
> Test Framework: Jest + ts-jest

## 1. Test Strategy

프롬프트 문자열 변경이므로, 단위 테스트로 프롬프트 내용이 올바른 키워드/구조를 포함하는지 검증한다. 기존 테스트 스위트의 회귀 테스트도 포함한다.

## 2. Test Environment

- Runtime: Node.js
- Framework: Jest 29 + ts-jest
- Dependencies: 없음 (추가 패키지 불필요)

## 3. Test Cases

### TC-01: CMD_TEAMS에 모드 판단 섹션 존재

- **Category**: Unit
- **Target**: CMD_TEAMS (teams.ts)
- **Preconditions**: teams.ts import
- **Steps**:
  1. CMD_TEAMS 문자열을 import
  2. "Small", "Medium", "Large" 키워드 존재 확인
  3. "모드 판단" 또는 "Pipeline Mode" 관련 텍스트 존재 확인
- **Expected Result**: 모든 키워드 포함
- **Edge Cases**: 없음

### TC-02: CMD_TEAMS에 모드별 실행 매트릭스 존재

- **Category**: Unit
- **Target**: CMD_TEAMS (teams.ts)
- **Steps**:
  1. CMD_TEAMS 문자열에서 "메인 직접" 키워드 확인
  2. "서브에이전트" 키워드 확인
  3. Step 1이 항상 "메인 직접"인지 확인
- **Expected Result**: 매트릭스 구조 포함

### TC-03: CMD_TEAMS_EN에 동일한 모드 구조 존재

- **Category**: Unit
- **Target**: CMD_TEAMS_EN (teamsEn.ts)
- **Steps**:
  1. CMD_TEAMS_EN에서 "Small", "Medium", "Large" 키워드 확인
  2. "Pipeline Mode" 텍스트 확인
  3. "main direct" 또는 "inline" 키워드 확인
- **Expected Result**: 영문 모드 구조 포함

### TC-04: CMD_TEAMS에 Step 1 인라인화 명시

- **Category**: Unit
- **Target**: CMD_TEAMS (teams.ts)
- **Steps**:
  1. "Step 1" 관련 텍스트에서 "메인" 또는 "직접" 키워드 확인
  2. req-manage 관련 서브에이전트 호출 지시가 없는지 확인
- **Expected Result**: Step 1은 메인 직접 실행으로 명시

### TC-05: 기존 테스트 회귀

- **Category**: Integration
- **Target**: 전체 테스트 스위트
- **Steps**:
  1. `npm test` 실행
  2. 모든 기존 테스트 통과 확인
- **Expected Result**: 0 failures

## 4. Coverage Goals

| Category | Target |
|----------|--------|
| Unit | CMD_TEAMS, CMD_TEAMS_EN 프롬프트 내용 100% |
| Integration | 기존 테스트 스위트 회귀 없음 |

## 5. Test Data

- CMD_TEAMS, CMD_TEAMS_EN 문자열을 직접 import하여 사용
