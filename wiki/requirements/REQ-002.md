# REQ-002: bugfix-teams 적응형 파이프라인 적용

> Date: 2026-03-10
> Status: [ ]

## 설명

/bugfix-teams 파이프라인에도 REQ-001에서 구현한 적응형 파이프라인(Adaptive Pipeline)을 적용한다. 버그 수정은 대부분 Small~Medium 규모이므로 서브에이전트 오버헤드 절감 효과가 크다.

## 주요 기능

### 1. bugfix-teams에 모드 판단 로직 추가

Step 1(bugfix — 원인 분석) 완료 후, bugfix 보고서의 영향 범위를 기반으로 Small/Medium/Large를 판단한다.

판단 기준은 /teams와 동일:
- 파일 변경 수: ≤3(Small), 4~10(Medium), >10(Large)
- 새 모듈 수: 0(Small), 1~2(Medium), ≥3(Large)
- UI 포함: 있으면 자동 Large

### 2. Step 1(bugfix) 모드별 분기

/teams와 달리 bugfix의 Step 1은 코드베이스 탐색이 필요하여 PRD 작성보다 무거울 수 있다:
- Small: 메인 직접 (단순 버그는 탐색 범위가 좁음)
- Medium/Large: 서브에이전트 (복잡한 원인 분석은 컨텍스트가 큼)

### 3. 모드별 실행 매트릭스

| Step | 스킬 | Small | Medium | Large |
|------|------|-------|--------|-------|
| 1 | /bugfix | 메인 직접 | 서브에이전트 | 서브에이전트 |
| 2 | /dev-impl | 메인 직접 | 메인 직접 | 서브에이전트 |
| 3 | /test-impl | 메인 직접 | 서브에이전트 | 서브에이전트 |
| 4 | /finalize | 메인 직접 | 메인 직접 | 서브에이전트 |

## 구현 대상

- `src/main/constants/commands/bugfixTeams.ts` — 파이프라인 프롬프트 수정
- `src/main/constants/commands/bugfixTeamsEn.ts` — 영문 버전 동기화

## 제약 사항

- 기존 스킬 파일(bugfix.ts, devImpl.ts, testImpl.ts, finalize.ts)은 변경하지 않는다
- /teams의 적응형 파이프라인 구조와 일관성을 유지한다

## 관련 요구사항

- REQ-001: 적응형 파이프라인 (Adaptive Pipeline) — 동일 패턴을 bugfix에 적용
