# REQ-001: 적응형 파이프라인 (Adaptive Pipeline) — 작업 크기 기반 서브에이전트 최적화

> Date: 2026-03-10
> Status: [ ]

## 설명

/teams 파이프라인에 작업 크기 판단 로직을 추가하여, 작업 규모에 따라 서브에이전트 사용을 최적화한다. 현재는 모든 작업에 동일하게 서브에이전트를 사용하여 작은 작업에서 오버헤드가 크고, 서브에이전트를 안 쓰면 큰 작업에서 TDD 사이클이 망가지는 문제가 있다.

## 주요 기능

### 1. 작업 크기 판단 (PRD 기반)

PRD 작성 후 SDD 복잡도를 기준으로 파이프라인 모드를 결정한다:

- **Small** (서브에이전트 0개 — 메인에이전트가 직접 실행)
  - 조건: 파일 변경 3개 이하, 새 모듈 0개, UI 없음
  - 예: 설정값 변경, 유틸 함수 1개 추가, 버그 수정
  - 실행: Step 1~4를 메인 에이전트가 순차 실행

- **Medium** (서브에이전트 1개 — TDD만 분리)
  - 조건: 파일 변경 4~10개, 또는 새 모듈 1~2개
  - 예: API 엔드포인트 추가, 기존 컴포넌트 확장
  - 실행: Step 1,2,4는 메인이 직접, Step 3(test-impl)만 서브에이전트

- **Large** (서브에이전트 3개 — 현재 방식)
  - 조건: 파일 변경 10개 초과, 또는 새 모듈 3개 이상, 또는 UI 포함
  - 예: 새 기능 전체 구현, 대규모 리팩토링
  - 실행: Step 2,3,4를 각각 서브에이전트로 실행

### 2. Step 1 (req-manage) 인라인화

PRD 작성은 항상 가벼운 작업(파일 읽기 + 문서 작성)이므로, 모든 모드에서 메인 에이전트가 직접 실행한다. 서브에이전트 1개 오버헤드를 즉시 제거.

### 3. Step 4 (finalize) 조건부 인라인화

빌드가 이미 Step 3에서 통과했다면, finalize는 wiki 파일 생성만 수행. Small/Medium 모드에서는 메인 에이전트가 직접 실행.

## 파이프라인 모드 매트릭스

| 모드 | Step 1 (req-manage) | Step 2 (dev-impl) | Step 3 (test-impl) | Step 4 (finalize) |
|------|---------------------|--------------------|--------------------|-------------------|
| Small | 메인 직접 | 메인 직접 | 메인 직접 | 메인 직접 |
| Medium | 메인 직접 | 메인 직접 | 서브에이전트 | 메인 직접 |
| Large | 메인 직접 | 서브에이전트 | 서브에이전트 | 서브에이전트 |

## 구현 대상

- `src/main/constants/commands/teams.ts` — 파이프라인 프롬프트 수정
- `src/main/constants/commands/teamsEn.ts` — 영문 버전 동기화

## 제약 사항

- TDD 사이클(Step 3)은 컨텍스트가 매우 커지므로, Medium 이상에서는 반드시 서브에이전트로 분리해야 한다
- 기존 스킬 파일(devImpl.ts, testImpl.ts, finalize.ts 등)은 변경하지 않는다 — teams.ts 프롬프트만 수정
- 작업 크기 판단은 메인 에이전트가 PRD 내용을 읽어 수행하며, 별도 코드 로직이 아닌 프롬프트 지시로 구현
