# SDD-0001: 적응형 파이프라인 (Adaptive Pipeline)

> Source: PRD-0001 / REQ-001
> Date: 2026-03-10
> Status: Draft

## 1. Overview

`/teams` 파이프라인의 프롬프트(CMD_TEAMS)를 수정하여, PRD 작성 후 작업 규모를 판단하고 Small/Medium/Large 모드에 따라 서브에이전트 사용을 최적화한다. 런타임 코드 변경 없이 프롬프트 문자열만 수정한다.

## 2. Architecture

현재 파이프라인:
```
Main Agent → Sub-agent(Step1) → Sub-agent(Step2) → Sub-agent(Step3) → Sub-agent(Step4)
```

변경 후:
```
Main Agent
  ├─ Step 1: 항상 메인 직접 (req-manage)
  ├─ [모드 판단: PRD 기반 Small/Medium/Large]
  ├─ Step 2: Small/Medium=메인 직접, Large=서브에이전트 (dev-impl)
  ├─ Step 3: Small=메인 직접, Medium/Large=서브에이전트 (test-impl)
  └─ Step 4: Small/Medium=메인 직접, Large=서브에이전트 (finalize)
```

## 3. Module Design

### 3.1 CMD_TEAMS 프롬프트 (teams.ts)

- **Responsibility**: 파이프라인 실행 지시를 포함하는 프롬프트 문자열
- **Location**: `src/main/constants/commands/teams.ts`
- **Dependencies**: 없음 (순수 문자열 상수)

#### 변경 사항

현재 프롬프트의 구조를 다음과 같이 수정한다:

**1) "서브에이전트 원칙" 섹션 → "실행 원칙" 섹션으로 교체**

기존:
```
## 서브에이전트 원칙 — 최소 컨텍스트 전달
...
```

변경:
```
## 실행 원칙

### Step 1 인라인화
Step 1(req-manage)은 항상 메인 에이전트가 직접 실행한다.

### 모드 판단
Step 1 완료 후 PRD 내용을 읽어 작업 규모를 판단한다:
- 영향 파일 수, 새 모듈 수, UI 존재 여부를 기준으로 분류

| 기준 | Small | Medium | Large |
|------|-------|--------|-------|
| 파일 변경 수 | ≤3 | 4~10 | >10 |
| 새 모듈 수 | 0 | 1~2 | ≥3 |
| UI 포함 | 없음 | 무관 | 있으면 자동 Large |

여러 기준 중 하나라도 상위 모드에 해당하면 상위 모드를 적용한다.
모드를 출력한다: "[Pipeline Mode: {mode}] ..."

### 서브에이전트 사용 규칙

서브에이전트 호출 시:
- 1~2줄 프롬프트: 스킬 이름 + 파일 경로 + 출력 번호만
- 파일 내용, 배경 설명, 워크플로 지시를 포함하지 않는다

메인 직접 실행 시:
- 해당 스킬의 워크플로를 메인 에이전트가 직접 수행한다
```

**2) "파이프라인 (4단계)" 섹션 → 모드별 매트릭스 추가**

기존 테이블 유지 + 모드별 실행 방식 매트릭스 추가:

```
## 파이프라인 모드별 실행

| Step | 스킬 | Small | Medium | Large |
|------|------|-------|--------|-------|
| 1 | /req-manage | 메인 직접 | 메인 직접 | 메인 직접 |
| 2 | /dev-impl | 메인 직접 | 메인 직접 | 서브에이전트 |
| 3 | /test-impl | 메인 직접 | 서브에이전트 | 서브에이전트 |
| 4 | /finalize | 메인 직접 | 메인 직접 | 서브에이전트 |
```

### 3.2 CMD_TEAMS_EN 프롬프트 (teamsEn.ts)

- **Responsibility**: CMD_TEAMS의 영문 번역 버전
- **Location**: `src/main/constants/commands/teamsEn.ts`
- **Dependencies**: 없음 (순수 문자열 상수)

#### 변경 사항

CMD_TEAMS와 동일한 구조 변경을 영문으로 적용한다.

## 4. Data Flow

1. 사용자가 `/teams` 실행
2. 메인 에이전트가 REQ 선택 → Step 1(req-manage) 직접 실행 → PRD 생성
3. 메인 에이전트가 PRD를 읽어 영향 파일 수, 새 모듈 수, UI 여부 판단
4. 모드(Small/Medium/Large) 결정 및 출력
5. 모드에 따라 Step 2~4를 메인 직접 또는 서브에이전트로 실행
6. 각 단계 검증 → 완료 처리

## 8. Edge Cases & Error Handling

| Scenario | Handling |
|----------|----------|
| PRD에서 파일 수를 판단하기 어려운 경우 | Medium으로 기본 적용 (안전한 중간값) |
| PRD에 UI 언급은 있으나 실제 HTML 목업이 불필요한 경우 | SDD에서 UI 섹션 Skip 명시하면 UI 없음으로 판단 |
| 기존 Large 모드와 동일한 동작 필요 시 | Large 모드 = 현재 동작과 완전히 동일 |
